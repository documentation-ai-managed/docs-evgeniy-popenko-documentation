## Get Started

Follow these steps to create and publish your first documentation page.

<Steps>
  <Step title="Create a New Page" icon="file-plus">
    Navigate to the content folder and add a new `{filename}.mdx` file. Start with YAML frontmatter for title and description.
  </Step>
  <Step title="Add Content" icon="edit-3">
    Use H2 headings and MDX components. Include code examples with realistic data.
  </Step>
  <Step title="Preview and Commit" icon="eye">
    Preview locally, then commit changes. Deploy automatically via GitHub Actions.

## Brand Guidelines

Maintain consistency across all documentation. Use the primary brand color `{#3B82F6}` for accents.

<Tabs>
  <Tab title="Web/CSS" icon="globe">
    Apply the brand color in your styles.

    ```css
    :root {
      --brand-color: #3B82F6;
    }

    .header {
      background-color: var(--brand-color);
      color: white;
    }
    ```
  </Tab>
  <Tab title="Icons and Typography" icon="type">
    Pair with clean sans-serif fonts like Inter or system defaults.

    - Headings: Bold, `{font-size: 1.5rem+}`
    - Body: `{line-height: 1.6}`
    - Icons: Lucide library, monochrome except accents
  </Tab>
  <Tab title="Dark Mode" icon="moon">
    Ensure readability in dark themes.

    ```css
    @media (prefers-color-scheme: dark) {
      --brand-color: #60A5FA;
    }
    ```
  </Tab>
</Tabs>
